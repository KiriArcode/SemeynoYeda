# SemeynoYeda — бриф для Figma Make

**Цель:** получить в Figma Make более яркий вариант дизайна с возможным переосмыслением элементов. Игнорировать текущие требования по дизайну — нужна только концепция и список функций. Деплой на Vercel позволяет добавить более сложные анимации и аккуратную работу с данными.

---

## Концепция продукта

**SemeynoYeda** — PWA для управления семейным питанием: планирование меню на неделю, рецепты с учётом диет, список покупок из меню, трекер морозилки, чек-листы и таймеры заготовок. Работает офлайн; данные синхронизируются с Supabase при наличии сети. Закрытый доступ — только по приглашению (семья).

**Проблема:** семьям с разными диетами сложно планировать неделю, учитывать заготовки в морозилке, собирать список покупок и организовывать день заготовок.

**Решение:** одно приложение — каталог рецептов с тегами и «для кого», недельное меню по приёмам пищи, автосписок покупок, морозилка со сроками, пошаговые планы и таймеры для заготовок.

**Пользователи:** семья из двух человек с разными диетическими потребностями (например, щадящее питание / обычное), общие закупки раз в неделю и общий день заготовок.

---

## Функции (перечень)

### 1. Рецепты
- CRUD рецептов (создание, просмотр, редактирование, удаление).
- Категории: основные блюда, соусы, гарниры, завтраки, полдники, супы, десерты.
- Теги: gastritis-safe, soft-texture, rich-feel, freezable, quick, prep-day, batch-cooking, overnight, packable, low-calorie, blanch-before-freeze, double-coating.
- Пометка «для кого»: один человек, второй, оба.
- Поля рецепта: название, подзаголовок, категория, теги, ингредиенты (количество + единицы), шаги приготовления, оборудование, время (подготовка / готовка / всего), хранение (холодильник, морозилка, вакуум), заметки.
- Поиск по названию в реальном времени.
- Фильтры: категории, теги, «для кого», комбинированные.
- Форма добавления и редактирования, валидация.
- Карточка в списке и детальная страница: ингредиенты, пошаговые инструкции, время, хранение, оборудование.
- Режим «из текста»: вставка сырого текста → парсинг → предзаполненная форма рецепта.

### 2. Меню на неделю
- Вид недели: 7 дней, горизонтальный скролл.
- Приёмы пищи: завтрак, обед, полдник, ужин (базово); опционально 5–6 приёмов для одного из пользователей.
- Слоты приёма: карточки блюд, пометка «для кого» (визуально).
- Замена блюда: выбор рецепта из каталога (модал/экран), фильтр по категории и «для кого».
- Добавление и удаление блюда в слоте, несколько блюд в одном слоте (для разных людей), вариации («с кабачковым соусом»).
- Предложения: по истории меню, по тегам/профилю, по разнообразию, с учётом морозилки.
- Детальный вид дня: все блюда, ссылки на рецепты, суммарное время, оборудование.

### 3. Профили
- Два профиля (например, Коля и Кристина) с настройками диеты.
- Настройки: применимые теги, текстуры, соусы, ограничения (что нельзя).
- Отдельный режим меню для одного профиля (5–6 приёмов).
- Фильтр «мои рецепты», учёт профиля в предложениях меню, визуальная индикация подходящих рецептов.

### 4. Список покупок
- Генерация из меню недели: сбор ингредиентов, суммирование, группировка по категориям (мясо, молочка, овощи, крупы и т.д.).
- Чекбоксы «куплено», ручное добавление, редактирование, удаление, очистка отмеченных.
- Группированный вид, привязка к рецептам, прогресс (сколько куплено), экспорт/шаринг.

### 5. Морозилка
- Добавление заготовки: рецепт, порции, дата заморозки; редактирование и удаление.
- Уменьшение порций при использовании.
- Расчёт срока годности по рецепту и дате заморозки, отображение оставшегося времени, уведомления о скором истечении, сортировка по сроку.
- Связь с меню: предложения из морозилки, отметка использования (уменьшение порций).
- Опционально: вес порции (г), калории и белок на порцию на карточке.

### 6. Заготовки (prep day / batch cooking)
- Список рецептов с тегом prep-day; выбор, что готовить.
- Пошаговый план: порядок шагов, параллельные процессы, оценки по времени, прогресс-бар.
- Таймеры на шаги, несколько таймеров параллельно, уведомления по окончанию.
- Batch cooking: выбор дня, фасовка (main + side), вакуумная упаковка, связь main/side для порций.

### 7. Режим готовки (cooking session)
- Выбор даты и приёма пищи, запуск сессии.
- Многошаговый сценарий: проверка ингредиентов → готовка по шагам → завершение.
- Параллельные таймеры во время готовки.
- Проверка наличия ингредиентов, кнопка «добавить в список покупок».

### 8. Настройки повара (chef mode)
- Включение/выключение режима повара (overlay или отдельный контекст).
- Настройки: время подготовки по умолчанию и прочие параметры готовки.

### 9. PWA и технический контекст
- Установка на устройство (manifest, иконки), standalone.
- Offline: Service Worker, кэш, работа с локальными данными без сети.
- Синхронизация с Supabase при наличии сети (offline-first).
- Адаптивность: mobile-first, touch targets не менее 44px, жесты при необходимости.

### 10. Навигация и оболочка
- Нижняя навигация: главные разделы (Меню, Рецепты, Морозилка, Покупки).
- Роутинг SPA, хлебные крошки, кнопка «Назад».
- Страницы: Меню, Рецепты (список + деталь + новый/редактирование), Покупки, Морозилка, Заготовки (prep), Готовка (cooking), Настройки повара.

### 11. Данные и интеграции (контекст для дизайна)
- Локальный кэш (IndexedDB) + Supabase как источник истины при онлайн.
- Закрытый доступ: только приглашённые пользователи.
- Опционально: выгрузка рецептов в Google Drive (один документ) для Notebook LM.

## Направление для Figma Make

- **Яркий вариант:** переосмыслить визуал — можно более светлый, контрастный или насыщенный, не привязываясь к текущей тёмной теме.
- **Элементы:** допускается переосмысление карточек, слотов меню, тегов, прогресс-баров, списков, таймеров — с целью ясности и выразительности.
- **Анимации и данные:** учитывать, что приложение на Vercel — возможны более сложные анимации (переходы между экранами, появление блоков, интерактивные состояния) и более наглядная подача данных (сводки, прогресс, сроки, статистика).
- Конкретные цвета, шрифты и компонентная библиотека не заданы — в приоритете концепция и полный перечень функций выше.
