# Промпт: Offline-First архитектура

## Контекст проекта

Я работаю над **SemeynoYeda** (https://semeyno-yeda.vercel.app/) — PWA-приложением с offline-first подходом. Архитектура: фронтенд работает с IndexedDB (Dexie), API (Vercel Serverless) работает с Neon PostgreSQL. Нужны материалы по стратегиям синхронизации.

## Задача

Найти материалы по **Offline-First архитектуре**:

- Стратегии синхронизации IndexedDB ↔ Neon PostgreSQL через API
- Разрешение конфликтов при синхронизации (last write wins, по updated_at)
- Оптимистичные обновления UI (сначала IndexedDB, затем push в API)
- Обработка ошибок в offline режиме
- Индикаторы синхронизации для пользователя

## Формат результатов

1. **Краткое описание** — что это и зачем нужно
